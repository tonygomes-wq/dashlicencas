PASSO 1: Restaurar Backup Antigo (que funcionava)
──────────────────────────────────────────────────

1. Você tem backup do index.html e assets/ que funcionavam?
   
   ✅ SIM → Restaure esses arquivos
   ❌ NÃO → Vá para PASSO 2 (Solução Alternativa)

2. Após restaurar, teste se o sistema volta a funcionar
   
3. Se funcionar, vá para PASSO 3


PASSO 2: Solução Alternativa (Se não tem backup)
─────────────────────────────────────────────────

Como o frontend está dando problema, vamos implementar APENAS o backend:

1. Mantenha o frontend antigo (que funciona)

2. Faça upload APENAS do backend:
   ✓ app_bitdefender_sync_client.php

3. Execute o SQL no banco:
   ✓ db_bitdefender_client_api.sql

4. O sistema volta a funcionar (sem a interface da API individual)

5. A API individual funcionará via requisições diretas ao PHP


PASSO 3: Atualizar Frontend Gradualmente (Opcional)

Quando o sistema estiver funcionando novamente:

1. Compile o frontend no ambiente de DESENVOLVIMENTO original
   (onde você desenvolve, não em produção)

2. Teste localmente primeiro (npm run dev)

3. Só depois faça o build e upload para produção

💡 COMO USAR A API INDIVIDUAL SEM INTERFACE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Mesmo sem a interface, você pode usar a funcionalidade:

1. Adicionar API Key via phpMyAdmin:
   
   UPDATE bitdefender_licenses 
   SET client_api_key = 'SUA_API_KEY_AQUI',
       client_access_url = 'https://cloud.gravityzone.bitdefender.com/api'
   WHERE id = 1;  -- ID do cliente

2. Sincronizar via requisição direta:
   
   POST para: /app_bitdefender_sync_client.php
   Body: {"clientId": 1}

3. Ou criar um botão simples no frontend antigo
